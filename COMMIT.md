# Contributing

These guidelines apply to everyone who contributes to the project. By submitting code, you agree to follow these standards to help maintain the quality, security, and integrity of the client. Contributions that may become part of an official release are expected to meet these requirements.

## Contribution Standards

**The following are not permitted:**

- Submitting plagiarized or copied code without proper authorization or attribution.
- Submitting code that has not been adequately tested.
- Submitting malicious, harmful, or intentionally disruptive code.
- Failing to provide appropriate credit for code, assets, or other work that you did not originally create.
- Submitting features that certify as hacking, exploits, or backdoors.
